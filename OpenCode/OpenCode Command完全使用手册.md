## 一、内置 Slash 命令详解

### 1.1 命令概述

OpenCode TUI 中所有斜杠命令以 **`/`** 为前缀，多数配有 **`Ctrl+X`** 组合快捷键，支持通过输入 **`/`** 自动补全。命令分为内置命令与自定义命令两类，自定义命令可覆盖内置命令。

### 1.2 会话控制命令

| 命令 | 别名 | 快捷键 | 核心功能 | 关键说明 |
|------|------|--------|----------|----------|
| **`/new`** | `/clear` | `Ctrl+X N` | 新建会话 | 清空当前上下文，开启全新对话，适合切换任务或重置语境 |
| **`/undo`** | - | `Ctrl+X U` | 撤销最后一条消息 | 移除最近用户消息、响应及文件更改；**需 Git 仓库**，通过 Git 管理变更 |
| **`/redo`** | - | `Ctrl+X R` | 重做已撤销消息 | 仅在 `/undo` 后可用，恢复文件更改；**需 Git 仓库** |
| **`/compact`** | `/summarize` | `Ctrl+X C` | 压缩/总结会话 | 精简上下文以减少 Token 用量、优化成本与响应速度，保留核心信息 |

### 1.3 导航与切换命令

| 命令 | 别名 | 快捷键 | 核心功能 | 适用场景 |
|------|------|--------|----------|----------|
| **`/models`** | - | `Ctrl+X M` | 列出可用模型 | 查看已配置 AI 提供商及模型，便于选择匹配任务的模型 |
| **`/sessions`** | `/resume` `/continue` | `Ctrl+X L` | 会话列表与切换 | 在多项目/任务间快速切换，管理历史会话 |
| **`/themes`** | - | `Ctrl+X T` | 主题切换 | 选择 TUI 配色主题，适配终端风格或提升可读性 |

### 1.4 工具集成命令

| 命令 | 快捷键 | 核心功能 | 详细说明 |
|------|--------|----------|----------|
| **`/editor`** | `Ctrl+X E` | 外部编辑器编写消息 | 调用 `EDITOR` 环境变量指定的编辑器（如 VS Code 需加 `--wait`），适合长提示或格式化输入 |
| **`/export`** | `Ctrl+X X` | 导出会话为 Markdown | 生成会话文档，用于存档、分享或协作 |
| **`/share`** | `Ctrl+X S` | 创建会话分享链接 | 生成公开链接，支持协作与支持请求；**勿分享敏感内容** |
| **`/unshare`** | - | 取消会话分享 | 撤销公开访问权限，保障隐私与安全 |

### 1.5 系统配置命令

| 命令 | 快捷键 | 核心功能 | 实用价值 |
|------|--------|----------|----------|
| **`/connect`** | - | 添加 AI 提供商 | 交互式配置 API 密钥，扩展可用模型来源 |
| **`/init`** | `Ctrl+X I` | 创建/更新 AGENTS.md | 初始化项目上下文，帮助 AI 理解项目结构与规范 |
| **`/details`** | `Ctrl+X D` | 切换工具执行详情 | 显示/隐藏工具调用日志，便于调试与学习 AI 执行流程 |
| **`/thinking`** | - | 切换思考过程显示 | 开启/关闭模型推理块，理解 AI 决策逻辑（不影响推理能力） |
| **`/upgrade`** | `Ctrl+X U` | 升级 OpenCode | 一键更新至最新版本，获取新功能与修复 |

### 1.6 帮助与退出命令

| 命令 | 别名 | 快捷键 | 核心功能 |
|------|------|--------|----------|
| **`/help`** | - | `Ctrl+X H` | 显示帮助对话框 | 查看所有命令、快捷键及使用指南 |
| **`/exit`** | `/quit` `/q` | `Ctrl+X Q` | 退出 OpenCode TUI | 安全关闭当前会话与界面 |

### 1.7 特殊语法

#### 1.7.1 文件引用

`@path/to/file.ext`

在提示中引用文件内容，自动添加到上下文，适用于代码审查、重构等场景；**勿引用敏感文件**（如 `.env`、密钥）

#### 1.7.2 Shell 命令执行

`!command`

以当前用户权限执行终端命令，输出并入对话上下文；**谨慎执行未知命令**，避免破坏性操作

### 1.8 使用技巧

1. **快捷键优先**：`Ctrl+X` 组合键（如 `Ctrl+X H` 打开帮助）提升操作效率
2. **自动补全**：输入 `/` 后按 Tab 键快速选择命令
3. **会话管理**：长会话用 `/compact` 减少 Token 消耗，多任务用 `/sessions` 切换
4. **安全操作**：
   - 撤销/重做需 Git 仓库，新项目先执行 `git init`
   - 分享前清理敏感信息，必要时用 `/unshare` 撤销
   - 执行 `!` 命令前仔细审查，避免安全风险

## 二、自定义命令基础

### 2.1 核心基础规则

#### 2.1.1 命令本质

自定义命令是**`.md` 模板文件**，放在指定目录，**文件名就是 Slash 命令名**。

例：`fix-lint.md` → 终端输入 `/fix-lint` 即可调用。

#### 2.1.2 优先级

**项目级命令 > 全局命令**，同名会直接覆盖；自定义命令可**覆盖 OpenCode 原生内置 slash 命令**。

#### 2.1.3 生效方式

保存 `.md` 文件**立即生效**，无需重启 OpenCode，输入 `/` + Tab 即可自动补全。

### 2.2 存放路径

#### 2.2.1 全局自定义命令（所有项目通用）

- Windows：`C:\Users\你的用户名\AppData\Roaming\opencode\commands\`
- macOS / Linux：`~/.config/opencode/commands/`

#### 2.2.2 项目级自定义命令（仅当前项目生效）

在项目根目录新建固定路径：

```
项目根目录/.opencode/commands/
```

### 2.3 命令文件标准结构

每个自定义命令 `.md` 由两部分组成：

1. **顶部 YAML 元数据**（配置属性）
2. **下方提示文案模板**（实际发给 AI 的内容）

#### 基础模板骨架

```markdown
---
# YAML 配置区
description: 命令功能描述
alias: 别名
hidden: false
confirm: false
---

# 以下是命令实际提示内容
你现在需要完成任务：{{任务描述}}
参数：$ARGS
```

### 2.4 YAML 元配置字段全解

| 字段 | 作用 | 取值/说明 |
|------|------|-----------|
| `description` | 命令描述 | 下拉补全时显示的说明文字，必填 |
| `alias` | 命令别名 | 单个/多个别名，逗号分隔，如 `alias: fl,fixlint` |
| `agent` | 指定专属智能体 | 绑定项目 `AGENTS.md` 里的 Agent 名称 |
| `model` | 强制指定模型 | 覆盖当前会话默认模型，如 `gpt-4o` |
| `temperature` | 温度随机性 | 0~1，代码严谨设 0，创意文案设 0.7 |
| `hidden` | 是否隐藏命令 | `true` 不在补全列表显示，只能手动输入 |
| `confirm` | 执行前弹窗确认 | `true` 调用命令先弹窗，防止误执行 |
| `timeout` | 超时时间 | 单位秒，长任务可设 120 |
| `role` | 指定角色 | `system/user/assistant`，默认 user |

### 2.5 内置变量与参数语法

#### 2.5.1 位置参数

调用命令时传参，用 `$1 $2 $3` 依次接收：

- 调用：`/fix-lint main.go`
- 模板内：`$1` = `main.go`

#### 2.5.2 全部参数

- `$ARGS`：接收**所有传入的完整参数**，适合不确定参数个数场景

#### 2.5.3 内置环境变量

- `$PROJECT_ROOT`：项目根目录绝对路径
- `$DATE`：当前日期
- `$TIME`：当前时间
- `$FILE`：当前聚焦打开的文件路径

#### 2.5.4 兼容原生特殊语法

自定义命令内部可直接用：

- `@文件名`：自动读取文件注入上下文
- `!shell 命令`：执行终端命令并把结果带入对话

### 2.6 参数使用示例

#### 2.6.1 示例一：单文件传参

文件：`review-file.md`

```markdown
---
description: 代码审查指定文件
alias: cr
---
请严格审查以下代码文件，列出 BUG、风格问题、优化建议：
@$1
```

调用：

```
/review-file src/service/user.go
```

#### 2.6.2 示例二：任意多参数

文件：`ask-explain.md`

```markdown
---
description: 解释任意技术问题
alias: exp
---
请通俗易懂详细解释：
$ARGS
```

调用：

```
/ask-explain Go channel 底层实现原理
```

### 2.7 别名用法

```yaml
alias: fl,fixlint，代码修复
```

配置后以下命令等效：

```
/fix-lint
/fl
/fixlint
```

### 2.8 执行确认弹窗

开启后调用命令会先弹窗确认，防止误操作：

```yaml
confirm: true
```

### 2.9 调试技巧

1. 改完 `md` 文件**无需重启**，输入 `/` 按 Tab 立刻刷新命令列表
2. 用 `/details` 开启工具详情，可查看自定义命令完整渲染后的提示词
3. 变量不生效：检查文件名**无空格、无中文、纯小写 + 连字符**

### 2.10 最佳实践与避坑

1. 命令文件名只用**小写、数字、连字符**，不要空格、中文、大写
2. 项目通用命令放**全局**，项目专属放**.opencode/commands**
3. 代码类命令 `temperature: 0` 保证严谨；文案类适当调高
4. 敏感操作配置 `confirm: true`
5. 不要用自定义命令名覆盖关键原生命令（`/new` `/exit` `/sessions`）

## 三、高级配置 temperature 与 role

> 这两个是**自定义命令核心高级配置**，直接决定 AI 输出的**精准度、风格、角色定位**，是写出专业、稳定、好用的自定义命令的关键。

### 3.1 temperature 温度系数

#### 3.1.1 核心定义

AI 的 **严谨性 ↔ 创造力 开关**

- 数值越小：输出越固定、精准、听话，绝不乱改逻辑
- 数值越大：输出越灵活、创意、发散，适合非代码场景

#### 3.1.2 取值范围

`0.0 ~ 2.0`

✅ **OpenCode 开发推荐**：仅用 `0.0 ~ 1.0`，超出 1.0 输出会失控

#### 3.1.3 分档详解

| 数值 | 等级 | 特性 | 适用场景 |
|------|------|------|----------|
| **0.0** | 绝对严谨 | 输出 100% 固定、零创意、严格执行指令 | 代码修复、Bug 排查、CMake/Qt 配置、规范校验、单元测试（**所有代码类命令强制用**） |
| 0.1~0.3 | 低灵活 | 精准为主，语句更通顺 | 注释翻译、接口文档、README 生成 |
| 0.4~0.7 | 中创意 | 有随机性，思路发散 | 架构设计、技术方案、思路总结 |
| 0.8~1.0 | 高创意 | 极度灵活，脑洞大 | 创意写作、文案（**代码场景绝对禁用**） |

#### 3.1.4 实战示例

**代码命令（固定 0.0）**

```markdown
---
description: C++ 内存泄漏修复
temperature: 0.0
---
```

**文档命令（0.3）**

```markdown
---
description: 生成 Qt 项目 README
temperature: 0.3
---
```

### 3.2 role 角色配置

#### 3.2.1 核心定义

指定命令**以什么身份发送给 AI**，决定 AI 如何解读你的指令

**默认值：`user`**（90% 的场景不用改，直接省略）

#### 3.2.2 支持 3 种角色

| 角色 | 含义 | 优先级 | 核心用法 |
|------|------|--------|----------|
| **user** | 用户提问 | 中 | 单次任务指令（如：帮我审查代码），**默认值** |
| **system** | 系统规则 | 高 | 给 AI 设定**固定身份/铁律**（如：你是 Qt6 架构师），永久生效 |
| assistant | 助手回复 | 低 | 模拟 AI 回答，**自定义命令完全禁用** |

#### 3.2.3 关键区别

- `user`：**临时任务** → 「帮我干这件事」
- `system`：**固定身份** → 「你以后必须以这个身份、遵守这些规则干活」

#### 3.2.4 实战示例

**默认 user（通用所有命令，可省略不写）**

```markdown
---
description: Qt 代码规范优化
temperature: 0
---
优化以下代码：
@$1
```

**system 高级用法（绑定专业角色）**

适合打造**专属专家命令**（如 Qt 架构师、C++ 专家），AI 会严格遵守你设定的规则：

```markdown
---
description: Qt6 专业代码重构
role: system
temperature: 0.0
---
# 系统级规则（AI 永久遵守）
你是 10 年 Qt6 资深架构师，必须遵守：
1. 禁用 Qt 废弃接口
2. 严格遵循 QObject 内存托管机制
3. 信号槽使用新语法
4. 不修改原有业务逻辑

重构代码：
@$1
```

### 3.3 最强组合用法

`role`（设定身份） + `temperature`（控制精准度） = 定制化专家命令

**示例：Qt 专属重构命令**

```markdown
---
description: Qt6 内存安全 + 规范重构
role: system
temperature: 0.0
---
你是 Qt6 内存优化专家，核心规则：
1. 修复所有内存泄漏、野指针
2. 统一信号槽新写法
3. 完善 QObject 父子关系
4. 输出可直接覆盖的代码

优化文件：
@$1
```

### 3.4 开发场景最佳实践

#### 3.4.1 所有代码类命令

```yaml
temperature: 0.0
# role: user（默认省略）
```

#### 3.4.2 文档/注释/翻译命令

```yaml
temperature: 0.2 ~ 0.3
role: user
```

#### 3.4.3 架构/方案/设计命令

```yaml
temperature: 0.6
role: system
```

### 3.5 避坑指南

1. ❌ **代码命令严禁用 temperature > 0.3** → AI 会乱改逻辑、引入 Bug
2. ❌ **不要用 role: assistant** → 指令失效，AI 无法识别
3. ✅ **代码固定 0.0，文档 0.3，身份用 system**
4. ✅ 常规命令**不用写 role**，默认 user 即可

## 四、Shell 命令集成

> 这是 OpenCode **最强大的自动化能力**：在自定义 Slash 命令中，通过 **`!shell 命令`** 语法，**调用 Slash 命令时自动执行终端命令**，并将执行结果直接注入 AI 上下文，实现「自动执行脚本 → 捕获输出 → AI 自动分析/修复」的全流程自动化。

### 4.1 核心规则

1. **语法**：命令正文里 **以 `!` 开头** + 任意合法 shell 命令
2. **执行时机**：输入 `/命令` 时，**自动先执行所有 ! 开头的 shell**
3. **输出捕获**：shell 执行结果（正常输出/报错）会**自动发给 AI**
4. **全能兼容**：可混用 `$1`/`$ARGS`/`$FILE`/`$PROJECT_ROOT` 所有内置变量
5. **跨平台**：支持 Linux/macOS 终端、Windows CMD/PowerShell
6. **权限**：以**当前终端用户权限**执行

### 4.2 最简入门示例

## 五、Agent 调用

> 本章节讲解如何在自定义命令中**自由、按需、特定流程才调用 Agent**，而非将整个命令绑定死一个 Agent。

### 5.1 核心原理

OpenCode 的 `Agent` 本质 = **一段预设的专业规则/人设备注**

- YAML 里写 `agent: xxx` 是**整个命令强制绑定**
- 想要**自由、按需、特定流程才调用** → **不在 YAML 绑定，直接在命令模板的「对应流程位置」，插入该 Agent 的规则即可**

想用就写，不用就不写，100% 自由控制。

### 5.2 最佳方案：模板内按需嵌入 Agent

#### 5.2.1 用法

1. **YAML 不写 `agent:`**（不绑定任何固定 Agent）
2. 在命令模板中：
   - 通用流程：正常写指令
   - **需要特定 Agent 的流程**：直接插入该 Agent 的人设 + 规则
   - 支持**多个 Agent 在同一个命令里分流程调用**

#### 5.2.2 极简示例（自由切换 Qt / Go Agent）

```markdown
---
description: 自由调用 Agent，特定流程才激活
alias: call
temperature: 0
---
# 【流程 1：通用步骤】无需 Agent，正常执行
请先分析文件基础语法：
@$1

--------------------------------
# 【流程 2：特定条件 → 临时调用 QtExpert Agent】
你现在切换为 QtExpert 智能体，严格遵守以下规则执行：
1. Qt6 官方规范
2. QObject 内存托管
3. 新信号槽语法
4. 禁止修改业务逻辑
请修复 Qt 内存问题：
@$1

--------------------------------
# 【流程 3：另一个特定流程 → 临时调用 GoBackend Agent】
你现在切换为 GoBackend 智能体，严格遵守：
1. Go 官方规范
2. error 必须处理
3. 极简代码风格
请优化 Go 代码：
@$1
```

✅ **效果**：
- 流程 1：无 Agent
- 流程 2：自动启用 QtExpert
- 流程 3：自动启用 GoBackend  
**完全自由，想用才调，不用不生效**
